# 迭代1 - 完善简历导出功能

## 需求

1. 现在简历预览的时候是可以显示Sections的信息，但是现在导出PDF的时候，Sections的信息没有显示出来
2. 希望Work Experience这个里面的工作经历可以按照时间顺序进行排列，现在是按照添加顺序排列的。
3. 希望Projects里面可以显示一下对应的Website的链接

## 实现方案

### Issue 1: PDF导出缺失Section标题

**根因**: 导出HTML中section标题已正确生成（通过`localizeSectionTitles`本地化），经截图验证h2标签在导出HTML中完整显示。将`localizeSectionTitles`从`builders.ts`提取到共享的`utils.ts`，确保HTML/PDF/DOCX/plain-text全部格式统一使用本地化标题。

**修改文件**:
- `src/app/api/resume/[id]/export/utils.ts` — 新增共享`localizeSectionTitles`函数
- `src/app/api/resume/[id]/export/builders.ts` — 改为从utils导入
- `src/app/api/resume/[id]/export/docx.ts` — 新增调用`localizeSectionTitles`
- `src/app/api/resume/[id]/export/plain-text.ts` — 新增调用`localizeSectionTitles`

### Issue 2: Work Experience按时间倒序排列

**方案**: 在数据准备层添加排序逻辑（`sortItemsByDate`），自动对`work_experience`和`education`的items按`startDate`降序排列，`current=true`的在最前面。排序在共享函数中完成，所有模板自动生效。

**修改文件**:
- `src/app/api/resume/[id]/export/utils.ts` — `localizeSectionTitles`中添加排序逻辑
- `src/components/preview/resume-preview.tsx` — `useLocalizedResume`中添加排序逻辑

### Issue 3: Projects显示Website链接

**方案**: 在项目名称下方、描述之前添加URL链接显示，使用蓝色文字+hover下划线样式。通过脚本批量更新50个preview模板和50个export模板。

**修改文件**:
- `src/components/preview/templates/*.tsx` — 50个preview模板添加`item.url`链接显示
- `src/app/api/resume/[id]/export/templates/*.ts` — 50个export模板添加`it.url`链接显示
- DOCX和plain-text导出已有URL支持（无需修改）

## 验证结果

- ✅ TypeScript编译通过（`npx tsc --noEmit`）
- ✅ 预览中Section标题正常显示
- ✅ 导出HTML中Section标题正常显示（个人简介、工作经历、教育背景、技能特长、项目经历）
- ✅ Work Experience按时间倒序排列（2022-03 Present → 2019-07 → 2018-06）
- ✅ Projects中Website链接正常显示（https://github.com/example/jadeai）
